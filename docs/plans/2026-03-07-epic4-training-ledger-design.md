# Epic 4 Story 4.1 — Training Ledger Schema Migration Design

> Date: 2026-03-07
> Status: Approved
> Reviewers: Winston (Architect), Amelia (Dev), Murat (QA), Bob (SM), John (PM)

## Problem

HOMS needs to track employee training compliance from LearnDash while allowing HR to override values without sync overwriting them. Compliance reporting must show effective values (HR overrides win over raw sync data).

## 3-Layer Compliance Model

| Layer | Table | Mutability | Who writes |
|-------|-------|-----------|------------|
| A (Raw) | `training_records` | Sync overwrites on each pull | LearnDash sync EF |
| B (Overrides) | `training_adjustments` | Append-only, immutable | HR via frontend |
| C (Effective) | `v_training_compliance` VIEW | Computed, not stored | Nobody — derived |

**Rule:** Effective value = latest Layer B override if exists, else Layer A value.

## Scale Assessment

- 50-200 employees per tenant, 10-30 courses
- 3-5 tenants at launch, scaling to 10
- Max ~60K training_records rows at full scale
- Regular Postgres VIEW handles this in single-digit ms — no materialized view needed

## Schema

### training_records (Layer A)

```sql
id              UUID PK DEFAULT gen_random_uuid()
tenant_id       UUID NOT NULL FK tenants(id)
person_id       UUID NOT NULL FK people(id)
course_id       TEXT NOT NULL          -- LearnDash course ID
course_name     TEXT                   -- human-readable, synced from LD
status          TEXT                   -- 'not_started' | 'in_progress' | 'completed'
completion_pct  INTEGER
completed_at    TIMESTAMPTZ            -- raw from LearnDash
training_hours  INTEGER                -- minutes, from LearnDash
expires_at      TIMESTAMPTZ            -- certification expiry (nullable, future use)
last_synced_at  TIMESTAMPTZ
created_at      TIMESTAMPTZ DEFAULT now()
updated_at      TIMESTAMPTZ DEFAULT now()
UNIQUE (tenant_id, person_id, course_id)
```

- RLS: own tenant (SELECT, INSERT, UPDATE)
- Sync pattern: `ON CONFLICT (tenant_id, person_id, course_id) DO UPDATE SET ...`
- Audit trigger: `audit_training_records()`

### training_adjustments (Layer B)

```sql
id              UUID PK DEFAULT gen_random_uuid()
tenant_id       UUID NOT NULL FK tenants(id)
person_id       UUID NOT NULL FK people(id)
course_id       TEXT NOT NULL
field           TEXT NOT NULL CHECK (field IN ('status', 'completion_pct', 'completed_at', 'training_hours'))
value           TEXT NOT NULL           -- override value (cast as needed)
reason          TEXT NOT NULL           -- required: why HR made this adjustment
actor_id        UUID NOT NULL FK auth.users(id)
created_at      TIMESTAMPTZ DEFAULT now()
```

- RLS: INSERT only for tenant_admin/hr_admin. SELECT own tenant. NO UPDATE. NO DELETE.
- Append-only — same pattern as `audit_log`
- Audit trigger: `audit_training_adjustments()`

### training_events (Immutable Audit Trail)

```sql
id              UUID PK DEFAULT gen_random_uuid()
tenant_id       UUID NOT NULL FK tenants(id)
person_id       UUID NOT NULL FK people(id)
course_id       TEXT
event_type      TEXT NOT NULL CHECK (event_type IN ('enrolled', 'completed', 'expired', 'adjusted'))
payload         JSONB
created_at      TIMESTAMPTZ DEFAULT now()
```

- RLS: INSERT only. SELECT own tenant. NO UPDATE. NO DELETE.
- Auto-generated by DB triggers on training_records and training_adjustments
- `expired` event type: schema-ready but detection deferred (no expiring courses yet)

### v_training_compliance (Layer C — VIEW)

Joins training_records with latest training_adjustments per (person_id, course_id, field). For each overridable field, uses `COALESCE(latest_adjustment, raw_value)`.

Returns one row per (person_id, course_id) with effective values for: status, completion_pct, completed_at, training_hours.

RLS inherited from underlying tables — no additional policy needed.

## Event Generation (Option A+)

DB triggers auto-generate training_events:

| Trigger on | Event type | Condition |
|-----------|-----------|-----------|
| training_records INSERT | `enrolled` | New record created |
| training_records UPDATE | `completed` | status changed to 'completed' |
| training_adjustments INSERT | `adjusted` | Any override inserted |
| (deferred) pg_cron | `expired` | expires_at < now() — built when needed |

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| VIEW type | Regular (not materialized) | 60K rows max, single-digit ms query time |
| field enum | CHECK constraint | Prevents typos; adding values is a one-line migration |
| training_hours | Added to both tables | LearnDash tracks time; HR may override |
| expires_at | Added to training_records | Future-proofs expiration detection without extra migration |
| expired detection | Deferred | No expiring courses today; schema supports it |
| Event generation | DB triggers | Guarantees no missed events regardless of code path |

## Migration File

Single migration: `20260307000001_epic4_training_ledger.sql`

Contents:
1. training_records table + RLS + audit trigger
2. training_adjustments table + RLS + audit trigger
3. training_events table + RLS (no audit trigger — it IS the audit trail)
4. Event-generating triggers (on training_records and training_adjustments)
5. v_training_compliance VIEW
